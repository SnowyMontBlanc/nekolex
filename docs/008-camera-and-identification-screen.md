# 008: カメラ & 識別画面

## 概要
カメラ撮影またはギャラリーから画像を選択し、Claude API で品種を識別して結果を表示する画面を実装する。

## 依存先
003 (ナビゲーション), 005 (UserContext), 007 (Claude API)

## ブロック対象
016

## 作業内容

### 1. カメラ画面 (`app/camera.tsx`)

#### 画面フロー
```
ホーム画面から遷移
  ↓
[📸 写真を撮る] / [🖼️ ギャラリーから選ぶ] 選択
  ↓
expo-image-picker で画像取得
  ↓
ローディング表示 (識別処理中...)
  ↓
結果表示
├─ 品種名 (日本語 + 英語)
├─ 信頼度 (%)
├─ 識別した特徴リスト
├─ ✨ 新発見！+10 XP (初回のみ)
└─ [図鑑で見る] ボタン → breed/[id] へ遷移
```

#### UI 構成
- 画像プレビュー (撮影/選択した画像)
- 撮影/選択ボタン (2つ並列)
- ローディングインジケーター
- 結果カード (品種名、信頼度、特徴)
- 新発見バナー (初回発見時のみ)
- アクションボタン (図鑑で見る / もう一度撮る)

### 2. カメラ権限ハンドリング
- expo-image-picker のパーミッション要求
- カメラ権限が拒否された場合の案内表示
- ギャラリー権限のハンドリング

### 3. 識別処理の実装
```typescript
// フロー
1. 画像取得 (expo-image-picker)
2. 画像リサイズ (image-processor.prepareImageForApi)
3. Claude API 呼び出し (claude-api.identifyBreed)
4. 結果を breeds.json と照合
5. 初回発見なら UserContext.discoverBreed() 呼び出し
6. 結果表示
```

### 4. 状態管理
- 画像 URI
- ローディング状態
- 識別結果 (IdentificationResult)
- エラー状態
- 新発見フラグ

### 5. エラー表示
- 猫が写っていない場合の案内
- API エラー時のリトライ案内
- ネットワークエラー時の案内

## 完了条件
- カメラ撮影で画像を取得できる
- ギャラリーから画像を選択できる
- Claude API で品種が識別される
- 識別結果が正しく表示される
- 初回発見時に XP が付与される
- 品種詳細画面へ遷移できる
- エラー時に適切なメッセージが表示される
